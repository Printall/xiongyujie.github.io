加载内核

exercise 4. 阅读有关C语言指针编程 *The C Programming Language

如果没有C语言基础的话，请不要跳过学习C语言指针问题。如果熟练掌握C语言，请跳过。

为了理解boot/main.c文件，需要了解ELF二进制文件。

当你编译C语言和链接一个C语言程序的时候，编译器将源文件”.c”文件输出到目标文件到”.o”文件，目标文件包括硬件期望的二机制格式的汇编语言指令。

连接器将所有已编译的目标文件组合到单个二进制映像中，例如ELF格式，代表可执行和可连接的。

elf格式可以参考：https://pdos.csail.mit.edu/6.828/2018/readings/elf.pdf。此格式不需要过深的了解。

对于6.828来说，将ELF可执行文件具有几个加载信息的头文件，然后是几个程序段，这几个程序段都是要指定加载到内存中的连续代码或者数据块。

ELF二进制文件以固定长度的ELF头开始，后面跟着可变长度的程序头，程序头列出每一个要加载的程序段。`

C语言再inc/elf.h定义了这些ELF头

在该程序中，我们感兴趣的是

  1. .text: 程序执行的指令
  2. .rodata: 只读数据
  3. .data: 程序初始化数据

当连接器计算程序的内存布局的时候，它同样保留了空间给未初始化的全局变量，C语言要求未初始化的全局变量以0开头。

检查内核可执行文件中所有部分的名称、大小、和链接完整地址。

objdump -h obj/keen/kernel

段的加载地址，是该段期望从中执行的内存地址，链接器以多种方式对二进制文件中的链接地址进行编码。

使用虚拟来解决位置依赖性

当你检查加载程序的链接地址和加载地址，他们匹配的十分完美。但是内核链接地址和加载地址存在相当大的差异。

链接内核更加的复杂，远远超出引导程序加载，所以链接和加载地址位于kern/kern.ld顶部，但是我没有看见这个文件

操作系统内核经常被链接和运行于很高的虚拟地址，例如0x00100000，为了释放低地址的空间给用户编程使用。

许多机器在0x00100000上没有实际的物理地址，所以我们不指望在其中存储内核。相反，我们将会使用进程空间来管理硬盘，通过映射虚拟地址0x00100000和物理地址0x00100000，使用这种方法，我们就可以留出足够大的用户进程空间了。

事实上，在下个实验，我们将映射256MB的内存，从0x00000000到0x0fffffff的物理空间到0x00000000到0xffffffff的虚拟空间。

现在我们开始映射4MB空间的内存，这已经足够让我们启动和运行。我们将在kern/entrypgdir.c手写和静态初始化页目录和页表，我们不需要了解这项工作的详情，只需要了解完成这项工作的含义。

在kern/entry.S摄入之CRO_PG标志位，CRO_PG一旦设置，内存引用就是虚拟地址，内存硬件就会将虚拟地址转换为物理地址。

训练9

确定内存在何处初始化堆栈，确定堆栈在内存中的位置。内核如何保留堆栈空间？栈堆指针指向保留区的哪一块地址？

加载程序引导内核之后，我们就进入了entry.S的相关内容，在这个文件中搜索stack，我们可以发现在69-77行的内容为初始化栈堆空间的。并且将栈顶指针bootstacktop存入寄存器esp的，我们可以在obj/kern/kerne.asm文件中发现该寄存器的值为0xf0110000。

​

内核为堆栈保留了一块空间，地址为0xf0110000，大小为8*4096·，堆栈指针指向保留区的高地址区。

  1. x86堆栈指针(esp寄存器)指向堆栈使用的最低区域。栈中压入一个值，则指针向下增长。弹出一个值，指针向上减少。在32位的模式下，栈堆只能保存32位的值，esp总是能被4整除。指令变量能够使用硬指针使用栈指针寄存器。
  2. ebp基本指针寄存器通过软件来于栈堆关联。在进入C语言的函数的时候，该函数的前言代码将会被压入栈，栈的信息通过栈顶指针来保存。在运行函数发的过程中，将esp的内容保存到ebp中。
